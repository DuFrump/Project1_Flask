기간: 2주

1. 최종 목표 및 시스템 구조 (Goal)
------------------------------------------------------------
- 핵심 목표: 사용자의 최초 발화 분석 후, 대화 주제를 '고정'하고, 사용자에게 그 주제에 관련된
	    활동이나 일정을 추천해준다. 해당 주제를 벗어나는 대화를 AI가 지능적으로 거절하도록
	    제어한다.

- 필수 구현 기능:
  1) NLP 분류 모델 (AI 파트): 채팅 로그를 정해진 주제로 정확하게 분류 (정확도 90% 이상 목표).
  2) 예측 확신도 기반 제어: 모델의 확신도에 따라 대화 흐름을 지능적으로 제어.
  3) 초기 5개의 질문을 통해, 주제의 신뢰도(Score)를 파악하고, 신뢰도 등수마다 점수를 부여,
     마지막에 가장 높은 2개의 주제를 사용자에게 추천
  4) 주제 이탈 감지 및 차단.
  5) 선택된 주제를 기준으로 사용자에게 활동 및 일정 생성
  6) 기존 대화 리스트 저장 기능


2. 팀원별 작업 분할 및 역할 (Tasks & Roles)
------------------------------------------------------------
| 담당자        | 주요 역할 (Core Task)                                        | 난이도  | 예측 소요 시간 |
|---------------|---------------------------------------------------------|---------|------------------|
| 지훈 (AI)      | 1. 주제 분류 모델 학습 및 최적화 (NLP/ML)              | 중상    | 4 ~ 5일          |
|                 | 2. 확신도 기반 제어 로직 설계 및 웹팀 전달 규격 정의 | 중상    | 2일               |
| 진석 & 다경 | 3. 웹 프론트엔드/백엔드 개발 (UI, 채팅 로직)             | 중      | 4 ~ 5일           |
|                 | 4. 백엔드에서 AI 모델과 LLM API 호출 로직 구현       | 중상    | 3일                |
| 지은           | 5. PPT 제작 및 최종 문서 정리                              | 중      | 2일                |
|                 | 6. 필요 데이터 생성 및 정제                                 | 하      | 2일                |

| 조장 (지훈)   | *조기 완료 시, 다른 팀원 작업 지원*                       | -       | -                   |


3. 성공을 위한 기술적 핵심 사항 (Critical Success Factors)
------------------------------------------------------------
1. 분류 모델 성능 확보:
   - **최소 90% 이상의 정확도**를 목표로 하며, TF-IDF를 활용한 지도 학습 모델(SVM, LR 등)을 우선 검토하여 명확한 키워드 기반 분류를 구현해야 함.

2. 주제 신뢰도 점수 로직 구현:
   - 초기 5개 질문에 대한 **모델의 예측 신뢰도**를 점수화하고, 백엔드에서 이 점수를 기반으로 **Top 2 주제를 추천**하는 비즈니스 로직을 명확히 정의해야 함.

3. 주제 이탈 감지 및 LLM 활용:
   - 주제 **'확정'** 후, 새로운 발화가 고정된 주제와 일치하지 않을 경우 NLP 모델이 **'Reject' 명령**을 반환해야 함.
   - 백엔드는 LLM API를 사용하여 "죄송하지만 [주제]에 집중해주세요."와 같이 **지능적으로 거절하는 응답**을 생성해야 함.

4. 데이터/API 규격 통일:
   - AI 파트와 웹 개발 파트 간의 데이터 전달은 **JSON** 포맷으로 통일해야 합니다.
   - 필수 필드: `{"topic": "영화", "confidence": 0.95, "action": "Confirm/Reject"}` 등을 명확히 정의하여 빠르고 정확한 통합 작업을 진행해야 함.

| 2주차 | 통합 및 최적화        | 분류 모델 성능 튜닝, LLM API와 백엔드 로직 통합, 확신도 기반 테스트, 최종 테스트 및 PPT 완성. |

!! 1, 2번 작업과 3, 4번 작업은 동시에 진행이 가능한가? !!
1. 전제 조건: 결과 규격의 확정 (Interface Contract)
두 그룹이 충돌 없이 병렬로 작업하기 위해서는 데이터를 주고받는 방식을 초기에 명확히 약속해야 합니다.

AI 파트 (지훈)의 약속: 지훈은 모델 학습이 끝나지 않았더라도, 자신의 분류 모델이 어떤 형식의 결과를 내보낼지 미리 확정하여 웹팀에게 알려줘야 합니다.

예시: {"주제": "영화", "확신도": 0.95}
웹 파트 (진석 & 다경)의 역할: 이 약속된 규격을 기반으로 UI와 백엔드 로직을 먼저 개발합니다. AI 모델이 아직 완성되지 않았다면, 임시로 **가짜 데이터(Dummy Data)**를 생성하여 통합 작업을 진행할 수 있습니다.

2. 순서 (Sequential)가 필요한 부분
4. 백엔드에서 AI 모델과 LLM API 호출 로직 구현 작업의 마지막 단계는 지훈의 AI 모델이 완성된 후에야 최종적으로 통합될 수 있습니다.

결론: 지훈은 1번 작업을 진행하는 동시에, 웹팀은 3번 작업을 시작하고 임시 데이터를 사용해 4번 작업의 뼈대를 구축할 수 있습니다. 이는 프로젝트 시간을 최소 4~5일 단축시키는 효과를 가져옵니다.
